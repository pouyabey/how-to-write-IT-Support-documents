# New Computer Deployment Checklist

## Purpose

The purpose of this checklist is to provide a simple and repeatable process for setting up a new computer or workstation for an employee or department.

This checklist helps IT staff prepare, configure, document, and verify a device before giving it to the user.

## Recommended Reference

A good starting point for this checklist is the Northwestern University FSM IT Computer Deployment Checklist. It provides a real IT example for computer deployment, including user information, device details, software, printers, mapped drives, data transfer, and final verification.

Northwestern University – FSM IT Computer Deployment Checklist:

## Pre-Setup Checklist

Before starting the setup, verify that:

- Device is available and ready for deployment
- Asset tag and serial number are recorded
- Device is added to inventory
- User account has been created
- Required software list is available
- Network, printer, and access requirements are confirmed

## Setup Checklist

Complete the following setup steps:

- Inspect the device for physical damage
- Connect required hardware and peripherals
- Connect the device to Wi-Fi or Ethernet
- Install or verify the correct operating system
- Run system updates
- Install required drivers
- Rename the computer using the organization naming standard
- Join the device to the domain or management platform if required
- Configure Microsoft 365, Outlook, Teams, and required applications
- Install antivirus or endpoint security software
- Enable disk encryption if required
- Add required printers and scanners
- Map required shared drives
- Configure VPN if required
- Test webcam, microphone, speakers, keyboard, and mouse

## Data Transfer

If replacing an old computer:

- Back up required files from the old device
- Transfer documents, desktop files, and required folders
- Transfer browser bookmarks if needed
- Transfer Outlook signatures if needed
- Verify transferred data with the user
- Document any files that could not be transferred

## Final Verification

Before giving the computer to the user, verify that:

- User can sign in successfully
- Internet connection works
- Email works
- Required software opens correctly
- Printers and scanners work
- Shared drives are accessible
- VPN works if required
- Device is updated
- Device is recorded in inventory
- User confirms the workstation is ready

## Troubleshooting / Common Issues

If there are problems during setup:

- If login fails, check account status, password, and group membership
- If network does not work, check Wi-Fi, Ethernet, IP address, DNS, and DHCP
- If shared drives are missing, check permissions and mapped drive settings
- If printers do not work, reinstall or remap the printer
- If software is missing, check the required software list
- If email does not sync, check Microsoft 365 license and Outlook profile
- If VPN does not work, verify credentials, MFA, and VPN client settings

## Notes

- Update this checklist whenever the deployment process changes
- Keep inventory records accurate
- Document setup issues in the ticketing system
- Escalate hardware, account, permission, or software issues when needed
- Verify the setup with the user before closing the ticket
